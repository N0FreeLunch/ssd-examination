# AI Agent Assets and Configuration

This directory (accessed via `.agents`) and the root-level configuration files define the behavior and assets for AI coding assistants working on this project.

## Symbolic Links to Specification Repository

To maintain a Single Source of Truth, all agent assets such as **Skills** and **Workflows** are stored within the `sdd-examination-spec` repository. The main project repository accesses these assets via symbolic links:

- `.agents/skills` -> `../sdd-examination-spec/.agents/skills`
- `.agents/workflows` -> `../sdd-examination-spec/.agents/workflows`

This ensures that any updates to the agent's domain knowledge or procedural workflows are versioned alongside the system specifications.

## White-list Based Version Control

A white-list approach is used in `.gitignore` to ensure that only the necessary symbolic links are tracked in this repository, preventing any dynamic state or unnecessary files within the `.agents` folder from being committed.

- `.agents/*` is ignored by default.
- `!.agents/skills` and `!.agents/workflows` are explicitly allowed (tracked).
